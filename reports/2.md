# 이슈 #2 — 차트 박스마크 zoom/pan 시 dot·label 위치 오류

## 요약

차트 확대/이동 시 박스권 고점·저점 dot과 label이 엉뚱한 위치에 표시되는 3가지 버그를 수정

---

## 원인

| # | 원인 | 증상 |
|---|------|------|
| 1 | X 범위 밖 label을 `Math.max(pad, ...)` 클램프 | 좌측 끝(x=40px)에 label 쌓임 |
| 2 | `priceToCoordinate` null 시 `chartHeight/2` 고정 | 모든 dot이 화면 중앙에 몰림 |
| 3 | 예측 series가 현재 뷰에 데이터 없어 `priceToCoordinate` 항상 null | 예측 박스 Y 좌표 무조건 null |

원인 3은 `[BOX_DIAG]` 진단 로그로 확인:
```json
{ "rawYHiOrig": null, "rawYLoOrig": null, "visiblePriceRange": null }
```

---

## 수정 내용 (`chart-render-overlays.js`)

### 1. 화면 밖 label hide 처리
```js
const hiOffScreen = (endPxHi !== null && endPxHi < 0) || (startPxHi !== null && startPxHi > overlayWidth);
const visibleHi = !hiOffScreen && rawTopHi >= 0 && rawTopHi <= chartHeight;
```

### 2. Y 좌표 둘 다 null이면 skip
```js
if (rawYHiOrig === null && rawYLoOrig === null) return;
```

### 3. series fallback으로 priceToCoordinate 해결
```js
function coordY(val) {
    const y = series.priceToCoordinate(val);
    if (y !== null) return y;
    for (const s of Object.values(chartState.seriesMap)) {
        if (s === series) continue;
        const fy = s.priceToCoordinate(val);
        if (fy !== null) return fy;
    }
    return null;
}
```
