아래 작업을 순서대로 진행해줘.

## 작업 목표
CLAUDE.md를 3가지 방향으로 개선한다.

---

## A. 단계별 게이트 방식 적용
이슈 처리 사이클 각 단계 끝에 ✅ 게이트 체크포인트를 추가해줘.
다음 단계로 넘어가려면 반드시 통과해야 하는 조건으로 명시할 것.

예시 형식:
> ✅ GATE: 린트·타입체크·테스트 통과 / context.md 업데이트 완료 → 다음 단계 진행

---

## B. 파일 분리
CLAUDE.md는 핵심 규칙과 흐름만 남기고,
아래 3개 파일로 세부 내용을 분리해줘:

- workflow_rules.md: 이슈 처리 사이클, 파일 탐색, 임시 파일, 보고서 규칙
- test_rules.md: 테스트 조건, 실행 순서, Mock 규칙, 커버리지 기준
- agent_roles.md: 에이전트 개입 시점, 전문 에이전트 역할 정의

CLAUDE.md 상단에는 각 파일 참조 안내를 추가할 것:
> 세부 규칙은 workflow_rules.md / test_rules.md / agent_roles.md 참조

---

## C. context.md 템플릿 생성
/temp/TEMPLATE/context.md 파일을 새로 만들어줘.
이슈 시작 시 이 템플릿을 복사해서 /temp/#이슈번호/context.md로 사용.

템플릿에 포함할 항목:
- [ ] 이슈 분석 완료
- [ ] 계획서 작성 완료
- [ ] 코드 수정 완료
- [ ] 린트·타입체크 통과
- [ ] 이슈 테스트 20개 통과 (커버리지 80% 이상)
- [ ] unit-tests/ 통과
- [ ] context.md 업데이트 완료
- [ ] 사양서 업데이트 판단 완료 (추가/유지/삭제)
- [ ] 보고서 저장 완료 (/reports/#N.md)
- [ ] gh issue comment 등록 완료
- [ ] PR 생성 및 머지 완료
