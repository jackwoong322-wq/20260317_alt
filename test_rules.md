# test_rules.md
> 테스트 규칙 전체 (순서, 조건, 금지사항)

---

## 실행 순서 (반드시 이 순서대로)

```
1. 단위 테스트       → jest [파일명]
2. 전체 테스트       → jest --passWithNoTests
3. 커버리지 측정     → jest --coverage
```

순서를 건너뛰거나 역순 실행 금지.

---

## 이슈 테스트 조건
- 이슈 수정 내용 기준 **20개** 작성
- 커버리지 **80% 이상** 달성 필수
- 실제 소스 import 필수 (로직 복사 금지)
- test.skip 사용 금지
- 외부 API 직접 호출 금지

---

## Mock 규칙
- Mock 선언 위치: **파일 상단** (테스트 케이스 내부 선언 금지)
- `beforeEach`에 `jest.clearAllMocks()` 필수
- Mock은 최소 범위만 적용 (불필요한 전체 모듈 Mock 금지)

---

## unit-tests/ 규칙
- unit-tests/ 내 파일 **수정 금지** (회귀 감지용 고정)
- unit-tests/ 실패 시 → 사이드이펙트 원인 분석 후 소스 수정
- 테스트끼리 상태 공유 금지 (전역 변수·파일 I/O 공유 금지)

---

## 금지사항 요약
| 금지 항목 | 이유 |
|-----------|------|
| 로직 복사 후 테스트 | 실제 소스 검증 불가 |
| test.skip | 미검증 코드 잔류 |
| API 직접 호출 | 외부 의존성으로 불안정 |
| unit-tests/ 수정 | 회귀 감지 기준 훼손 |
| 테스트 간 상태 공유 | 순서 의존성·오염 발생 |
| Mock을 테스트 내부 선언 | 범위 불명확·중복 선언 |

---

## 커버리지 기준
- 라인 커버리지: **80% 이상**
- 미달 시 추가 테스트 케이스 작성 후 재측정
- 커버리지 리포트는 context.md 테스트 섹션에 기록
